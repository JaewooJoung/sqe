Autonomous Truck Regulations & Standards Framework

Note: This document compiles current regulations and standards as of May 2025. Regulatory frameworks for autonomous trucks are evolving rapidly, and updates should be verified with the relevant agencies.

Agency Regulation/Initiative Status Key Points Reference

3. ISO Standards for Autonomous Trucks

Standard Title Application Key Points Reference
ISO/TS 5083 Road vehicles — Safety for automated driving systems — Design, verification and validation Trucks and buses
  • Safety framework for ADS design and testing
  • Excludes systems in operation prior to publication
 ISO
ISO 34503 Road Vehicles – Test Scenarios for Automated Driving Systems L3/L4 ADS
  • Rules for organizing ADS functioning information
  • Structured classification for defining Operational Design Domain (ODD)
  • Primarily designed for L3 and L4 ADS
 Invensity
ISO 39003 Road traffic safety — Guidance on ethical considerations relating to safety for autonomous vehicles L5 vehicles
  • Ethical guidance for AV design and operation
  • Not prescriptive about decision-making outcomes
  • Applicable to vehicles in L5 mode according to SAE J3016
 ISO
ISO 22737 Intelligent transport systems — Low-speed automated driving systems for predefined routes L4 systems
  • First international safety standard for L4 automated driving
  • Performance requirements and test procedures
  • Primarily for low-speed operations on predefined routes
 ADAS & AV International
ISO 21448 SOTIF (Safety Of The Intended Functionality) All ADS
  • Addresses safety risks not covered by ISO 26262
  • Focuses on performance limitations and misuse scenarios
 Automotive IQ
ISO 26262 Functional safety — Road vehicles All vehicles
  • Base functional safety standard
  • Needs extension for autonomous applications
  • Limitations when applied to machine learning systems
 Automotive IQ

4. SAE J3016 Automation Levels

Level Name Definition Driver/System Responsibilities Application Status Reference
0 No Driving Automation Zero autonomy Driver performs all driving tasks Conventional vehicles SAE
1 Driver Assistance Single function automation
  • System: Either steering OR acceleration/braking
  • Driver: All other aspects of driving
 Widely available (cruise control, lane keeping) ANSI Blog
2 Partial Driving Automation Combined function automation
  • System: Both steering AND acceleration/braking
  • Driver: Must monitor environment and be ready to take control
 Available in many vehicles (Tesla Autopilot, GM Super Cruise) Engineer's Vault
3 Conditional Driving Automation Limited self-driving
  • System: All driving tasks within ODD
  • Driver: Must be ready to intervene when requested
 Limited availability (Mercedes Drive Pilot) CMU User Guide
4 High Driving Automation Full self-driving under specific conditions
  • System: All driving tasks within ODD including fallback
  • Driver: Not needed within ODD
 Testing phase for trucks, limited deployment SAE
5 Full Driving Automation Full self-driving under all conditions
  • System: All driving tasks in all conditions
  • Driver: Never needed
 Theoretical, not yet achieved SAE

6. Relationships Between Frameworks

Relationship Type Description Impact on Development Reference
SAE Levels → Federal Regulations Regulations reference SAE levels to define requirements Different requirements apply to different automation levels NHTSA
ISO Standards → Testing Requirements ISO standards define testing scenarios and methods Compliance requires meeting ISO test procedures Invensity
Federal → State Regulations Federal frameworks may eventually preempt state regulations Currently requires compliance with both levels Roll Call
ODD Definition (ISO) → Operational Limits ISO 34503 defines ODD structure that limits where L4 can operate Directly impacts where L4 trucks can legally operate Invensity
Functional Safety (26262) → SOTIF (21448) Base safety standard extended by SOTIF for autonomous functions Need to address both standards for complete safety case Semi Engineering
